Really basic introduction to svn

Metaphor

The key concept is that the documents are in a database: the official latest documents version is in an online database and users have local copies. A user work cycle consists of: updating the local copy of the documents, editing the local copy, commiting the local copy back to the database. Multiple users can edit at the same time the same documents and svn will do as good a job as possible at merging. The main svn website is: http://www.subversion.org

Unix (Linux/Mac OS X) Instructions

  • install an "svn client" on your computer (you do not need the server, only the client). Example clients are available wikipedia svn clients.
  • ask Francesco for a userid and password
  • create your own local working copy of the documents by typing
    svn checkout https://fbsvn.me.ucsb.edu/svn/XXX --username francesco
    where you should substitute your userid for "francesco", and your desired directory for XXX. This instruction creates the directory "XXX" with all relevant files inside.
  • Each time you start working on the file(s), you should get the latest version by typing inside the XXX directory:
    svn update
  • if the local file is updated, you MUST reload the file in your editor before making any changes
  • The local svn database is manipulated as if it was a unix directory:
    • to add a file (or a directory) to the database, say svn add filename
    • to create a directory in the database, say svn mkdir dirname
    • to remove a file from the database, say svn rm filename
    These commands and editing files only change the "local version" of the database.
  • Each time you want to propagate your local changes to the main database, you should commit your changes back to the database by typing inside the XXX directory
    svn commit -m "type a message here"
  • Look at the output of the "svn commit" command. In particular, look for conflict warnings. If there is a conflict, then you must resolve it before re-committing the changes.
    • edit the file in question (e.g., New.bib or main.tex), look for the markes <<<<<< ===== >>>>>> and choose the right version.
    • type svn resolved main.tex
    • type svn commit -m "resolved conflict"

Final quick: to get html file to show properly on a browser, type 
svn propset svn:mime-type text/html *.html
or for pdf files: 
svn propset svn:mime-type application/pdf *.pdf
